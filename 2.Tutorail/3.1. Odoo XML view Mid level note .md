# Odoo View → Action → Menu → User (Complete Mental Model)

```text
                models.py
                    │
                    ▼
            student.student
                    │
                    ▼
         ir.ui.view (View Layer)
      ┌─────────┬─────────┬─────────┬────────┐
      │ Tree    │ Form    │ Kanban  │ Graph  │
      │ Search  │ Calendar│ Pivot   │ Activity│
      └─────────┴─────────┴─────────┴────────┘
                    │
                    ▼
      ir.actions.act_window (Action)
                    │
                    ▼
          ir.ui.menu (Menu)
                    │
                    ▼
              User clicks Menu
                    │
                    ▼
          Odoo opens the requested View
```

# Step 3 : Action

Action-এর কাজ

> কোন Model খুলবে?
>
> কোন কোন View Available হবে?
>
> কোন View Default হবে?

Example

```xml
<record id="student_action"
        model="ir.actions.act_window">

    <field name="name">Students</field>

    <field name="res_model">student.student</field>

    <field name="view_mode">tree,form</field>

</record>
```

মানে

```

↓

Available View

Form

↓

Default

Tree
```

# View Inheritance (Most Important Concept)

Odoo-র ৯০% Standard Module-এ তুমি **নতুন View তৈরি করো না**, বরং Existing View Modify করো।

Base View

```
student_form_view
```

```
Name

Age
```

এখন তুমি শুধু Email যোগ করতে চাও।

তখন পুরো Form Copy করো না।

বরং

```xml
<record id="student_form_inherit"
        model="ir.ui.view">

    <field name="inherit_id"
           ref="student_form_view"/>

    <field name="arch"
           type="xml">

        <xpath expr="//field[@name='age']"
               position="after">

            <field name="email"/>

        </xpath>

    </field>

</record>
```

ফলাফল

```
Original

Name

Age
```

↓

Inheritance Apply

↓

```
Name

Age

Email
```

Original View Change হয়নি।

শুধু Extend হয়েছে।

# Odoo Source Code পড়ার সময় Cheat Sheet

| XML দেখলে                                     | বুঝবে                                                                  |
| --------------------------------------------- | ---------------------------------------------------------------------- |
| `<record model="ir.ui.view">`                 | নতুন View তৈরি বা Inherit করা হচ্ছে                                    |
| `<field name="model">student.student</field>` | এই View কোন Model-এর                                                   |
| `<field name="inherit_id" ref="..."/>`        | Existing View Modify করা হচ্ছে                                         |
| `<xpath ...>`                                 | Original View-এর নির্দিষ্ট জায়গায় পরিবর্তন করা হচ্ছে                 |
| `<record model="ir.actions.act_window">`      | Action তৈরি হচ্ছে                                                      |
| `<field name="res_model">`                    | কোন Model খুলবে                                                        |
| `<field name="view_mode">`                    | কোন ধরনের View (tree, form, kanban...) ব্যবহার হবে এবং প্রথমটি Default |
| `<field name="view_id">`                      | নির্দিষ্ট একটি Default View ব্যবহার করো                                |
| `<field name="views" eval="...">`             | একাধিক নির্দিষ্ট View-এর Order ও Reference Explicitly Define করা       |
| `<menuitem action="...">`                     | Menu থেকে কোন Action চালু হবে                                          |
| `ref="..."`                                   | XML ID Reference করা হচ্ছে                                             |

# Golden Rules (Interview + Real Project)

1. **একটি Model-এর জন্য অনেক View থাকতে পারে।**
2. **একটি Action একটি Model-কে Open করে।**
3. **`view_mode` শুধু View-এর Type ও Default Order বলে (`tree,form,kanban`), নির্দিষ্ট View নয়।**
4. **`view_id` একটি নির্দিষ্ট Default View Force করতে ব্যবহৃত হয়।**
5. **`views` ব্যবহার করে একাধিক নির্দিষ্ট View ও তাদের Order Explicitly Control করা যায়।**
6. **একটি Menu সবসময় একটি Action Call করে।**
7. **Advanced Odoo Development-এ বেশিরভাগ সময় নতুন View লেখার চেয়ে Existing View `inherit_id` + `xpath` দিয়ে Extend করা হয়।**
8. **`ref="..."` বা XML ID হলো Odoo-এর "Primary Key" for XML Objects—View, Action, Menu, Report, Security Rule—সবকিছুকে একে অপরের সাথে যুক্ত করে।**

> **মনে রাখার একটি লাইন:** **Menu** কখনও View খোলে না, **View** কখনও Database-এ যায় না। **Menu → Action → View → Model → Database** — এটাই Odoo UI-এর মূল Execution Flow।
