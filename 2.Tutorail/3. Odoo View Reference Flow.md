তুমি যদি **Odoo MVC flow** বুঝতে চাও, তাহলে **View → Action → Menu** এই ৩টা জিনিস সবচেয়ে গুরুত্বপূর্ণ।

---

# পুরো Flow

```text
models.py
     │
     ▼
Student Model
     │
     ▼
student_view.xml
(Tree/Form/Search View)
     │
     ▼
Action (ir.actions.act_window)
     │
     ▼
Menu (ir.ui.menu)
     │
     ▼
Browser
```

---

# Step 1 : Model

```python
from odoo import models, fields

class Student(models.Model):
    _name = "student.student"

    name = fields.Char()
    age = fields.Integer()
```

এখানে Model-এর নাম

```text
student.student
```

---

# Step 2 : View

View শুধু বলে

**কিভাবে দেখাবে।**

```xml
<record id="student_tree_view" model="ir.ui.view">

    <field name="name">student.tree</field>

    <field name="model">student.student</field>

    <field name="arch" type="xml">

        <tree>
            <field name="name"/>
            <field name="age"/>
        </tree>

    </field>

</record>
```

এখানে

```text
model="student.student"
```

মানে

এই View কোন Model-এর।

---

Form View

```xml
<record id="student_form_view" model="ir.ui.view">

    <field name="model">student.student</field>

    <field name="arch" type="xml">

        <form>

            <sheet>

                <group>

                    <field name="name"/>
                    <field name="age"/>

                </group>

            </sheet>

        </form>

    </field>

</record>
```

# Step 3 : Action

এখন প্রশ্ন হচ্ছে

**User Menu-তে click করলে কোন View খুলবে?**

এটার উত্তর Action।

```xml
<record id="student_action"
        model="ir.actions.act_window">

    <field name="name">Students</field>

    <field name="res_model">student.student</field>

    <field name="view_mode">tree,form</field>

</record>
```


মানে

কোন Model Open হবে।

```
view_mode
```

মানে

কোন View Mode ব্যবহার হবে।

```
tree,form
```

মানে

প্রথমে List View

তারপর

Form View

# Step 4 : Menu

# কেন View ID Action-এ লেখা হয়নি?

অনেকেই এখানে Confused হয়।

Action-এ দেখো

```xml
<field name="view_mode">
tree,form
</field>
```

Tree View-এর ID নেই।

কারণ Odoo নিজেই Default Tree/Form View খুঁজে নেয়।

যদি একই Model-এর একাধিক Tree View থাকে, তখন নির্দিষ্ট View দিতে পারো:

```xml
<field name="view_id" ref="student_tree_view"/>
```

# Odoo Scaffolding-এ Folder Structure

```
student_management/

│
├── models/
│     └── student.py
│
├── views/
│     └── student_view.xml
│
├── security/
│     └── ir.model.access.csv
│
└── __manifest__.py
```

`student_view.xml`-এ সাধারণত এই ক্রমে লেখা হয়:

```xml
<odoo>

    <!-- Views -->
    <record id="student_tree_view" model="ir.ui.view">
        ...
    </record>

    <record id="student_form_view" model="ir.ui.view">
        ...
    </record>

    <!-- Action -->
    <record id="student_action"
            model="ir.actions.act_window">
        ...
    </record>

    <!-- Menu -->
    <menuitem
        id="student_root_menu"
        name="Student"/>

    <menuitem
        id="student_menu"
        name="Students"
        parent="student_root_menu"
        action="student_action"/>

</odoo>
```

## সহজে মনে রাখার ট্রিক

```
Model
   │
   ▼
View
   │
   ▼
Action
   │
   ▼
Menu
   │
   ▼
User Click
```

* **Model** → Data কোথায় আছে।
* **View** → Data কীভাবে দেখাবে।
* **Action** → কোন Model এবং কোন View খুলবে।
* **Menu** → User কোথা থেকে Action চালাবে।

এই Flow একবার পরিষ্কার হয়ে গেলে Odoo-তে নতুন Module তৈরি, View যোগ করা, বা Menu/Action কনফিগার করা অনেক সহজ হয়ে যায়।
