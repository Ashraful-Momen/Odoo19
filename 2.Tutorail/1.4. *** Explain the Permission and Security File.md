Odoo-তে `security/ir.model.access.csv` ফাইলটি **Access Control List (ACL)** হিসেবে কাজ করে। এটি নির্ধারণ করে **কে (Group)** কোন **Model**-এর উপর কী ধরনের permission পাবে।

তোমার CSV:

```csv
id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink

student_user,
student.user,
model_student_student,
base.group_user,
1,
1,
1,
1
```

এখন প্রতিটি field সহজভাবে ব্যাখ্যা করি।

| Column        | Meaning                           | তোমার Example           |
| ------------- | --------------------------------- | ----------------------- |
| `id`          | এই ACL rule-এর unique XML ID      | `student_user`          |
| `name`        | Human readable name               | `student.user`          |
| `model_id:id` | কোন Model-এর জন্য rule            | `model_student_student` |
| `group_id:id` | কোন User Group এই permission পাবে | `base.group_user`       |
| `perm_read`   | Read/View করতে পারবে?             | `1` = Yes               |
| `perm_write`  | Edit করতে পারবে?                  | `1` = Yes               |
| `perm_create` | নতুন Record Create করতে পারবে?    | `1` = Yes               |
| `perm_unlink` | Delete করতে পারবে?                | `1` = Yes               |

# এই Model কোথা থেকে আসে?

ধরো তোমার Model:

```python
from odoo import models, fields

class Student(models.Model):
    _name = "student.student"
    _description = "Student"

    name = fields.Char()
    age = fields.Integer()
```

এখানে

```python
_name = "student.student"
```

Odoo automatically একটি XML ID তৈরি করে

```
model_student_student
```

Rule:

```
student.student

↓

dot(.) replace হবে underscore(_)

↓

student_student

↓

আগে model_

↓

model_student_student
```

আর তাই CSV-তে লিখি

```csv
model_student_student
```

# `id` এবং `name`-এর পার্থক্য

অনেকেই এই দুইটি নিয়ে বিভ্রান্ত হয়।

```csv
id,name
student_user,student.user
```

`id`

* Technical XML ID।
* Module-এর ভিতরে unique হতে হবে।
* অন্য XML ফাইল থেকে reference করা যায়।

`name`

* শুধু Human-readable label।
* Backend-এ ACL-এর নাম হিসেবে দেখা যায়।
* Unique হওয়া বাধ্যতামূলক নয়।

# সম্পূর্ণ CSV

```csv
id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink
student_user,student.user,model_student_student,base.group_user,1,1,1,1
```

এর অর্থ:

* **Rule ID:** `student_user`
* **Rule Name:** `student.user`
* **Model:** `student.student`
* **Applicable Group:** `base.group_user` (Internal Users)
* **Permissions:** Read ✔, Write ✔, Create ✔, Delete ✔

> **গুরুত্বপূর্ণ:** `ir.model.access.csv` শুধু **model-level access** নিয়ন্ত্রণ করে। যদি তুমি নির্দিষ্ট কিছু record (যেমন একজন শিক্ষক শুধু নিজের ছাত্রদের দেখতে পারবে) নিয়ন্ত্রণ করতে চাও, তাহলে `security/*.xml`-এ **Record Rules (`ir.rule`)** ব্যবহার করতে হবে। ACL বলে *"কোন model ব্যবহার করা যাবে?"*, আর Record Rule বলে *"সেই model-এর কোন কোন record দেখা বা পরিবর্তন করা যাবে?"*।
