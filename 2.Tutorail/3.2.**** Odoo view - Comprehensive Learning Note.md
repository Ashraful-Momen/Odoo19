## 1. Core Concept

### The Golden Rule (Golden Rule)

**Every Odoo screen follows this EXACT pattern. No exceptions.**

```
┌──────────┐
│   MENU   │  "I am just a button. Click me, I'll fire an Action."
└────┬─────┘
     │  action="some_action_id"
     ▼
┌──────────┐
│  ACTION  │  "I decide: which Model? which Views? what Context?"
│ir.actions│
│.act_window│
└────┬─────┘
     │  res_model="some.model"
     │  view_mode="list,form,kanban"
     │  context="{'key':'value'}"
     ▼
┌──────────┐
│  MODEL   │  "I am the database table. I hold the data."
│_name=    │
│some.model│
└────┬─────┘
     │
     ▼
┌──────────┐
│  VIEWS   │  "I display the data. Multiple views per action."
│  - List  │
│  - Form  │
│  - Kanban│
│  - Graph │
│  - Pivot │
│  - Search│
└──────────┘
```

### Analogy (Analogy)

Think of it like a **restaurant**:

| Odoo Component | Restaurant Equivalent |
|:---|:---|
| **Menu** | Menu card item (e.g., "Pizza") — just a name, can't be eaten |
| **Action** | Waiter — takes your order, knows which kitchen (model) and how to serve (view) |
| **Model** | Kitchen — where the actual food (data) is prepared |
| **View** | Plate & presentation — how the food looks when served |
| **Context** | Special instructions — "extra cheese", "no onions" |

## 2. Custom Addon Structure

### Full File/Folder Tree

```
my_school/                          ← Addon root folder
│
├── __manifest__.py                 ← Addon metadata & dependencies
│
├── models/                         ← Python: Database tables & business logic
│   ├── __init__.py
│   └── student.py                  ← Model definition: school.student
│
├── views/                          ← XML: All UI definitions
│   ├── __init__.py                 ← (usually empty, but keep for import)
│   ├── menu.xml                    ← Menu items only
│   ├── action.xml                  ← Window actions
│   └── student_views.xml           ← Form, List, Search, Kanban views
│
├── security/                       ← Access rights
│   └── ir.model.access.csv
│
├── data/                           ← Demo/seed data (optional)
│   └── demo_students.xml
│
└── static/                         ← CSS, JS, images (optional)
    └── description/
        └── icon.png
```

### `__manifest__.py`

```python
{
    'name': 'School Management',
    'version': '1.0',
    'category': 'Education',
    'summary': 'Manage students, classes, and teachers',
    'depends': ['base', 'mail'],          # Dependencies
    'data': [
        'security/ir.model.access.csv',   # Load order matters!
        'views/menu.xml',                 # Menu first
        'views/action.xml',               # Then actions
        'views/student_views.xml',        # Then views
    ],
    'installable': True,
    'application': True,                  # Shows in Apps menu
}
```

## 3. Step-by-Step: Building Each Component

### Step 1: Create the Model (`models/student.py`)

```python
from odoo import models, fields

class SchoolStudent(models.Model):
    _name = 'school.student'              # Database table name
    _description = 'School Student'
    _rec_name = 'name'                    # Field used for display name

    name = fields.Char(string='Student Name', required=True)
    roll_number = fields.Char(string='Roll Number')
    class_id = fields.Many2one('school.class', string='Class')
    phone = fields.Char(string='Phone')
    email = fields.Char(string='Email')
    state = fields.Selection([
        ('draft', 'Draft'),
        ('enrolled', 'Enrolled'),
        ('graduated', 'Graduated'),
        ('dropped', 'Dropped'),
    ], string='Status', default='draft')
```

### Step 2: Create Menus (`views/menu.xml`)

```xml
<?xml version="1.0" encoding="utf-8"?>
<odoo>

    <!-- ========== ROOT MENU (Top-level in sidebar) ========== -->
    <menuitem
        id="school_menu_root"
        name="🏫 School"
        web_icon="my_school,static/description/icon.png"
        sequence="10"/>

    <!-- ========== CHILD MENU (Sub-menu, no action = just a folder) ========== -->
    <menuitem
        id="school_student_menu"
        name="📚 Students"
        parent="school_menu_root"/>

    <!-- ========== LEAF MENU (Has action = clickable, opens a screen) ========== -->
    <menuitem
        id="menu_school_all_students"
        name="👨‍🎓 All Students"
        parent="school_student_menu"
        action="action_school_all_students"/>

    <menuitem
        id="menu_school_enrolled_students"
        name="✅ Enrolled Students"
        parent="school_student_menu"
        action="action_school_enrolled_students"/>

</odoo>
```

**Sidebar rendering:**

```
🏫 School                    ← school_menu_root
   │
   └── 📚 Students           ← school_student_menu (just a folder, no action)
         │
         ├── 👨‍🎓 All Students    ← menu_school_all_students (clickable!)
         │
         └── ✅ Enrolled Students ← menu_school_enrolled_students (clickable!)
```

> **Rule:** A menu item WITHOUT `action=""` is just a folder/category. WITH `action=""` it opens a screen.

### Step 3: Create Actions (`views/action.xml`)

```xml
<?xml version="1.0" encoding="utf-8"?>
<odoo>

    <!-- ========== ACTION 1: Show ALL students ========== -->
    <record id="action_school_all_students" model="ir.actions.act_window">
        <field name="name">All Students</field>
        <field name="res_model">school.student</field>
        <field name="view_mode">list,kanban,form,graph,pivot</field>
        <field name="search_view_id" ref="school_student_search_view"/>
        <field name="help" type="html">
            <p class="o_view_nocontent_smiling_face">
                Add your first student!
            </p>
        </field>
    </record>

    <!-- ========== ACTION 2: Show ONLY enrolled students ========== -->
    <record id="action_school_enrolled_students" model="ir.actions.act_window">
        <field name="name">Enrolled Students</field>
        <field name="res_model">school.student</field>
        <field name="view_mode">list,form</field>
        <!-- MAGIC: Context & Domain filter -->
        <field name="domain">[('state', '=', 'enrolled')]</field>
        <field name="context">{'default_state': 'enrolled'}</field>
    </record>

    <!-- ========== ACTION 3: Same model, DIFFERENT default view ========== -->
    <record id="action_school_student_kanban" model="ir.actions.act_window">
        <field name="name">Student Board</field>
        <field name="res_model">school.student</field>
        <field name="view_mode">kanban,list,form</field>
    </record>

</odoo>
```

**Action fields explained:**

| Field | Purpose | Example |
|:---|:---|:---|
| `res_model` | Which database table to query | `school.student` |
| `view_mode` | Which view types are available (first = default) | `list,kanban,form` |
| `domain` | Hard filter (user CANNOT remove) | `[('state','=','enrolled')]` |
| `context` | Default values & behavior flags | `{'default_state':'enrolled'}` |
| `search_view_id` | Which search/filter bar to use | `ref="my_search_view"` |
| `help` | Empty-state message | HTML content |

### Step 4: Create Views (`views/student_views.xml`)

```xml
<?xml version="1.0" encoding="utf-8"?>
<odoo>

    <!-- ==================== LIST VIEW (Tree) ==================== -->
    <record id="school_student_list_view" model="ir.ui.view">
        <field name="name">school.student.list</field>
        <field name="model">school.student</field>
        <field name="arch" type="xml">
            <tree>
                <field name="name"/>
                <field name="roll_number"/>
                <field name="class_id"/>
                <field name="phone"/>
                <field name="state" widget="badge"/>
            </tree>
        </field>
    </record>

    <!-- ==================== FORM VIEW ==================== -->
    <record id="school_student_form_view" model="ir.ui.view">
        <field name="name">school.student.form</field>
        <field name="model">school.student</field>
        <field name="arch" type="xml">
            <form>
                <sheet>
                    <div class="oe_title">
                        <h1>
                            <field name="name" placeholder="Student Name..."/>
                        </h1>
                    </div>
                    <group>
                        <group string="Personal Info">
                            <field name="roll_number"/>
                            <field name="phone"/>
                            <field name="email"/>
                        </group>
                        <group string="Academic Info">
                            <field name="class_id"/>
                            <field name="state" widget="statusbar"/>
                        </group>
                    </group>
                </sheet>
            </form>
        </field>
    </record>

    <!-- ==================== SEARCH VIEW ==================== -->
    <record id="school_student_search_view" model="ir.ui.view">
        <field name="name">school.student.search</field>
        <field name="model">school.student</field>
        <field name="arch" type="xml">
            <search>
                <!-- Search fields (user types here) -->
                <field name="name"/>
                <field name="roll_number"/>
                <field name="phone"/>

                <!-- Pre-built filters (one click) -->
                <filter
                    name="enrolled"
                    string="✅ Enrolled"
                    domain="[('state','=','enrolled')]"/>

                <filter
                    name="draft"
                    string="📝 Draft"
                    domain="[('state','=','draft')]"/>

                <!-- Group by (organize records) -->
                <group expand="0" string="Group By">
                    <filter
                        name="group_by_class"
                        string="Class"
                        context="{'group_by':'class_id'}"/>
                    <filter
                        name="group_by_state"
                        string="Status"
                        context="{'group_by':'state'}"/>
                </group>
            </search>
        </field>
    </record>

    <!-- ==================== KANBAN VIEW ==================== -->
    <record id="school_student_kanban_view" model="ir.ui.view">
        <field name="name">school.student.kanban</field>
        <field name="model">school.student</field>
        <field name="arch" type="xml">
            <kanban class="o_kanban_mobile">
                <field name="state"/>
                <templates>
                    <t t-name="kanban-box">
                        <div class="oe_kanban_card">
                            <div class="oe_kanban_content">
                                <field name="name"/>
                                <field name="roll_number"/>
                                <field name="class_id"/>
                                <field name="state" widget="badge"/>
                            </div>
                        </div>
                    </t>
                </templates>
            </kanban>
        </field>
    </record>

</odoo>
```

### Step 5: How Odoo Auto-Discovers Views

**You DO NOT need to manually attach views to actions!** Odoo does it automatically by matching `model`:

```
Action: res_model = "school.student", view_mode = "list,form,kanban"
                │
                │  Odoo searches: "Which views have model='school.student'?"
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  List        Form       Kanban
  View        View        View
  (tree)      (form)     (kanban)
```

**Priority:** Lower priority number = higher priority (used first).

```xml
<field name="priority" eval="10"/>   <!-- Higher priority (used first) -->
<field name="priority" eval="99"/>   <!-- Lower priority (fallback) -->
```

## 4. View Inheritance Deep Dive

### Concept: Primary View vs Inherited View

```
┌──────────────────────────┐
│  PRIMARY VIEW (parent)   │  ← Original definition, id="school_student_form_view"
│                          │
│  <form>                  │
│    <field name="name"/>  │
│    <field name="phone"/> │
│  </form>                 │
└──────────┬───────────────┘
           │  inherit_id="school_student_form_view"
           ▼
┌──────────────────────────┐
│  INHERITED VIEW (child)  │  ← Adds/modifies parent, id="school_student_form_extended"
│                          │
│  <field name="email"     │     Adds email field AFTER phone
│    position="after"      │
│    ...                   │
│  />                      │
└──────────────────────────┘
```

### Inheritance Positions

```xml
<record id="school_student_form_extended" model="ir.ui.view">
    <field name="name">school.student.form.extended</field>
    <field name="model">school.student</field>
    <field name="inherit_id" ref="school_student_form_view"/>
    <field name="arch" type="xml">

        <!-- position="before": Add BEFORE the target field -->
        <field name="phone" position="before">
            <field name="roll_number"/>
        </field>

        <!-- position="after": Add AFTER the target field -->
        <field name="name" position="after">
            <field name="nickname"/>
        </field>

        <!-- position="inside": Add INSIDE a container (group, sheet, div) -->
        <group position="inside">
            <group string="Extra Info">
                <field name="blood_group"/>
            </group>
        </group>

        <!-- position="replace": Replace the target entirely -->
        <field name="phone" position="replace">
            <field name="phone" widget="phone"/>
        </field>

        <!-- position="attributes": Modify attributes only -->
        <field name="name" position="attributes">
            <attribute name="placeholder">Full Name...</attribute>
            <attribute name="required">1</attribute>
        </field>

    </field>
</record>
```

### Visual: Inheritance Positions

```
<form>                               <!-- position="inside" (on <form>) -->
  <sheet>                            <!-- position="inside" (on <sheet>) -->
    BEFORE  ←────────────────── position="before"
    [field name="phone"]       ← target field
    AFTER   ←────────────────── position="after"
    <group>                          <!-- position="inside" (on <group>) -->
      ...
    </group>
    REPLACE ←────────────────── position="replace" (replaces whole field)
  </sheet>
</form>
```

### XPath Inheritance (More Flexible)

When you can't target by field name, use XPath:

```xml
<field name="arch" type="xml">
    <xpath expr="//field[@name='name']" position="after">
        <field name="nickname"/>
    </xpath>

    <!-- Add after the first group inside sheet -->
    <xpath expr="//sheet/group[1]" position="after">
        <group string="New Section">
            <field name="extra_info"/>
        </group>
    </xpath>

    <!-- Target by CSS class -->
    <xpath expr="//div[hasclass('oe_title')]" position="inside">
        <h3>Subtitle Here</h3>
    </xpath>
</field>
```

## 5. Search Views & Filters

### Complete Search View Anatomy

```xml
<record id="my_search_view" model="ir.ui.view">
    <field name="name">my.model.search</field>
    <field name="model">my.model</field>
    <field name="arch" type="xml">
        <search>

            <!-- ① Search fields: user can type here -->
            <field name="name" string="Search by Name"/>
            <field name="email"/>
            <field name="phone"/>

            <!-- ② Pre-built filters: one-click apply -->
            <filter
                name="my_active_filter"         <!-- Internal ID -->
                string="✅ Active Records"       <!-- Display label -->
                domain="[('active','=',True)]"   <!-- Domain applied -->
                help="Show only active records"/> <!-- Tooltip -->

            <!-- ③ Date filters -->
            <filter
                name="created_today"
                string="Created Today"
                domain="[('create_date','>=',datetime.datetime.now().strftime('%Y-%m-%d 00:00:00'))]"/>

            <!-- ④ Separator (visual divider) -->
            <separator/>

            <!-- ⑤ Group By: organize into categories -->
            <group expand="0" string="Group By">
                <filter
                    name="group_by_user"
                    string="User"
                    context="{'group_by':'user_id'}"/>

                <filter
                    name="group_by_date"
                    string="Creation Date"
                    context="{'group_by':'create_date:month'}"/>
                    <!-- :day, :week, :month, :quarter, :year -->
            </group>

            <!-- ⑥ Favorites: user can save custom filters here -->
            <!-- (automatically available, no XML needed) -->

        </search>
    </field>
</record>
```

### Search View Visual

```
┌─────────────────────────────────────────────────────────┐
│  🔍 Search...  │ [Name ▾] [Email ▾]                     │  ← search fields
│                                                         │
│  Filters: [✅ Active] [Today] [My Records] │ Group By ▾ │  ← filters + group by
│                                                         │
│  ★ Favorites: [My Custom Filter] [Save current]         │  ← favorites
└─────────────────────────────────────────────────────────┘
```

## 6. Context & Domain Magic

### Domain: Server-Side Filter (Hard)

```xml
<!-- Hard filter: user CANNOT remove this -->
<field name="domain">[('state', '=', 'enrolled')]</field>
```

**Common domain operators:**

| Operator | Meaning | Example |
|:---|:---|:---|
| `=` | Equals | `[('state','=','done')]` |
| `!=` | Not equals | `[('state','!=','cancel')]` |
| `in` | In list | `[('state','in',('draft','sent'))]` |
| `not in` | Not in list | `[('state','not in',('done',))]` |
| `>`, `<` | Greater/Less | `[('amount','>',100)]` |
| `like` | Contains | `[('name','like','school')]` |
| `ilike` | Case-insensitive | `[('name','ilike','SCHOOL')]` |
| `&` | AND (default) | `['&',('a','=',1),('b','=',2)]` |
| `\|` | OR | `['\|',('a','=',1),('b','=',2)]` |

**Complex domain example:**

```xml
<!-- Show: enrolled students OR draft students with phone number -->
<field name="domain">[
    '|',
        ('state', '=', 'enrolled'),
        '&',
            ('state', '=', 'draft'),
            ('phone', '!=', False)
]</field>
```

### Context: Pass Data & Behavior

```xml
<field name="context">{
    'default_state': 'enrolled',     <!-- Pre-fill state field -->
    'default_user_id': uid,          <!-- Pre-fill with current user -->
    'search_default_enrolled': 1,    <!-- Auto-apply "enrolled" filter -->
    'group_by': 'class_id',          <!-- Auto group by class -->
    'create': False,                 <!-- Disable Create button -->
    'delete': False,                 <!-- Disable Delete button -->
    'form_view_ref': 'my_module.custom_form_view',  <!-- Force specific form view -->
}</field>
```

### The SAME model, DIFFERENT menus trick

This is the most powerful pattern in Odoo:

```
                    ┌─────────────────────────┐
                    │    school.student       │
                    │    (One database table)  │
                    └───────────┬─────────────┘
                                │
            ┌───────────────────┼───────────────────┐
            ▼                   ▼                   ▼
    ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
    │ ACTION:        │   │ ACTION:        │   │ ACTION:        │
    │ All Students   │   │ Enrolled       │   │ Graduated      │
    │                │   │                │   │                │
    │ domain: []     │   │ domain:        │   │ domain:        │
    │ (no filter)    │   │ [state=        │   │ [state=        │
    │                │   │  enrolled]     │   │  graduated]    │
    └───────┬───────┘   └───────┬───────┘   └───────┬───────┘
            │                   │                   │
            ▼                   ▼                   ▼
    ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
    │ VIEW:          │   │ VIEW:          │   │ VIEW:          │
    │ student_list   │   │ student_list   │   │ student_list   │
    │ (same view!)   │   │ (same view!)   │   │ (same view!)   │
    └───────────────┘   └───────────────┘   └───────────────┘
```

```xml
<!-- Menu 1: All Students -->
<menuitem id="menu_all" action="action_all_students"/>

<record id="action_all_students" model="ir.actions.act_window">
    <field name="name">All Students</field>
    <field name="res_model">school.student</field>
    <field name="view_mode">list,form</field>
    <!-- NO domain = show all -->
</record>

<!-- Menu 2: Enrolled Only -->
<menuitem id="menu_enrolled" action="action_enrolled"/>

<record id="action_enrolled" model="ir.actions.act_window">
    <field name="name">Enrolled Students</field>
    <field name="res_model">school.student</field>
    <field name="view_mode">list,form</field>
    <field name="domain">[('state','=','enrolled')]</field>
    <!-- SAME model, SAME views, DIFFERENT filter! -->
</record>
```

```text
│       │
│       ├── 👨‍🎓 All Students             menu_school_all_students
│       │   │                           action="action_school_all_students"
│       │   │
│       │   └──► action_school_all_students  [ir.actions.act_window]
│       │           │
│       │           ├── res_model = school.student
│       │           ├── view_mode = list,kanban,form,graph,pivot
│       │           ├── domain = []  (show everything)
│       │           └── context = {}
│       │                   │
│       │                   └──► school.student  [Model]
│       │                           │
│       │                           └──► Views found by model match:
│       │                               ├── school_student_list_view   [tree]
│       │                               ├── school_student_form_view   [form]
│       │                               ├── school_student_kanban_view [kanban]
│       │                               └── school_student_search_view [search]
│       │
│       └── ✅ Enrolled                  menu_school_enrolled_students
│           │                           action="action_school_enrolled_students"
│           │
│           └──► action_school_enrolled_students  [ir.actions.act_window]
│                   │
│                   ├── res_model = school.student  ← SAME model!
│                   ├── view_mode = list,form        ← Fewer view types
│                   ├── domain = [('state','=','enrolled')]  ← FILTER!
│                   └── context = {'default_state': 'enrolled'}
│                           │
│                           └──► school.student  ← SAME model, SAME views!
│                               (but domain filters to enrolled only)
```

## 8. Cheat Sheet

### ir.actions.act_window Fields

| Field | Type | Description |
|:---|:---|:---|
| `name` | char | Action display name |
| `res_model` | char | **Target model (required)** |
| `view_mode` | char | Comma-separated: `list,form,kanban,graph,pivot,calendar,activity` |
| `domain` | char | Server-side filter `"[('field','op','value')]"` |
| `context` | char | Session context `"{'key':'value'}"` |
| `search_view_id` | many2one | Specific search view to use |
| `view_id` | many2one | Force a specific view (bypass priority) |
| `view_ids` | one2many | Attach specific views with `ir.actions.act_window.view` |
| `limit` | integer | Records per page (default 80) |
| `target` | selection | `current` (main area) or `new` (popup dialog) |
| `help` | html | Empty state message |
| `binding_model_id` | many2one | Auto-create menu on this model |
| `binding_view_types` | char | For which view types to show the button |

### ir.actions.act_window.view (Explicit View Attachment)

When you need to EXACTLY control which view is used (bypassing priority matching):

```xml
<record id="action_orders_with_custom_views" model="ir.actions.act_window">
    <field name="name">Orders</field>
    <field name="res_model">sale.order</field>
    <field name="view_mode">list,form</field>
</record>

<!-- Attach a SPECIFIC list view -->
<record id="action_orders_list_view" model="ir.actions.act_window.view">
    <field name="act_window_id" ref="action_orders_with_custom_views"/>
    <field name="view_id" ref="my_custom_list_view"/>
    <field name="view_mode">list</field>
    <field name="sequence" eval="1"/>
</record>

<!-- Attach a SPECIFIC form view -->
<record id="action_orders_form_view" model="ir.actions.act_window.view">
    <field name="act_window_id" ref="action_orders_with_custom_views"/>
    <field name="view_id" ref="my_custom_form_view"/>
    <field name="view_mode">form</field>
    <field name="sequence" eval="2"/>
</record>
```

### View Types Reference

| `view_mode` string | XML Tag | ir.ui.view `type` |
|:---|:---|:---|
| `list` | `<tree>` | `tree` |
| `form` | `<form>` | `form` |
| `kanban` | `<kanban>` | `kanban` |
| `graph` | `<graph>` | `graph` |
| `pivot` | `<pivot>` | `pivot` |
| `calendar` | `<calendar>` | `calendar` |
| `activity` | `<activity>` | `activity` |
| `search` | `<search>` | `search` (not in view_mode) |

### Common Mistakes & Solutions

| Mistake | Why It Fails | Fix |
|:---|:---|:---|
| View doesn't show up | `model` in view ≠ `res_model` in action | Make them match exactly |
| Menu doesn't appear | Missing `security/ir.model.access.csv` | Add access rights |
| Field not found in view | Field not in model's `_name` | Check model Python file |
| Filter not working | Wrong domain syntax | Test in Odoo developer mode |
| Inherited view not applying | Wrong `inherit_id` reference | Verify parent view exists |
| Menu visible but empty | No records + no `help` message | Add `<field name="help">` or create records |

### Reference (`ref`) Explained

```xml
<!-- ref="module_name.xml_id" -->
<!-- Odoo translates this to database ID at runtime -->

<field name="inherit_id" ref="school_student_form_view"/>
<!-- → "Find the record with xml_id='school_student_form_view' in DB,
       use its database ID as the parent view." -->

<field name="view_id" ref="my_module.some_view"/>
<!-- → Cross-module reference: module_name.xml_id -->
```

### When to Use What

```
Need a new screen?            → Menu + Action + (optional) new View
Need to ADD fields?           → Inherited View (position="after"/"before")
Need to CHANGE existing UI?   → Inherited View (position="attributes"/"replace")
Need to REMOVE fields?        → Inherited View (position="replace" with nothing)
Need different filter?        → New Action with `domain`
Need pre-filled values?       → Action with `context` (default_xxx keys)
Need auto-applied filter?     → Action with `context` (search_default_xxx keys)
Need same data, different UI? → New Action, SAME model, DIFFERENT `view_id`
```

### File 2: `models/library_book.py`

```python
from odoo import models, fields

class LibraryBook(models.Model):
    _name = 'library.book'
    _description = 'Library Book'

    name = fields.Char('Title', required=True)
    author = fields.Char('Author')
    isbn = fields.Char('ISBN')
    state = fields.Selection([
        ('available', 'Available'),
        ('borrowed', 'Borrowed'),
        ('lost', 'Lost'),
    ], default='available')
```

### File 3: `security/ir.model.access.csv`

```csv
id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink
access_library_book_user,library.book.user,model_library_book,base.group_user,1,1,1,1
```
