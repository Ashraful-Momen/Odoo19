এটাই Odoo শেখার সবচেয়ে গুরুত্বপূর্ণ অংশ। একটা Module-এর প্রতিটি File-এর নির্দিষ্ট দায়িত্ব (Responsibility) আছে। যদি এটা বুঝে যাও, তাহলে Odoo-এর ৭০% Architecture পরিষ্কার হয়ে যাবে।

---

# Odoo Module Flow

```text
                    User Click
                        │
                        ▼
                 Browser (Frontend)
                        │
                        ▼
                  Controller (Optional)
                        │
                        ▼
                   Model (Python)
                        │
             Create / Read / Update / Delete
                        │
                        ▼
                  PostgreSQL Database
                        ▲
                        │
                   XML View (UI)
                        │
                        ▼
                  User sees Result
```

# 1. `__manifest__.py`

## এটা কী?

এটা Module-এর **Identity Card** বা **Configuration File**।

Odoo প্রথমে এই ফাইল পড়ে বুঝে:

* Module-এর নাম কী?
* Version কত?
* Author কে?
* কোন Module-এর উপর নির্ভর করে?
* কোন XML Load করবে?
* কোন Security Load করবে?

---

### উদাহরণ

```python
{
    "name": "Student Management",
    "version": "1.0",
    "depends": ["base"],
    "data": [
        "security/ir.model.access.csv",
        "views/student_view.xml",
    ],
    "installable": True,
}
```

---

### ASCII Flow

```text
Odoo Start
     │
     ▼
Read __manifest__.py
     │
     ├── Module Name
     ├── Dependencies
     ├── XML Files
     ├── Security
     └── Version
```

---

# 2. `__init__.py`

## এটা কী?

Python-এর Import File।

Odoo নিজে `models.py` খুঁজে পায় না।

তুমি `__init__.py` দিয়ে বলে দাও কোন Python File Import করতে হবে।

---

### Root `__init__.py`

```python
from . import models
from . import controllers
```

মানে

```text
student_management

↓

models Folder

↓

controllers Folder
```

---

### models/**init**.py

```python
from . import models
```

মানে

```text
models Folder

↓

models.py
```

---

### Flow

```text
__init__.py

↓

Import models

↓

Import controllers

↓

Python Class Ready
```

# 3. `models/models.py`

## এটা কী?

এখানে **Business Logic** লেখা হয়।

Database Table, Field, Method—সব এখানে থাকে।

---

### উদাহরণ

```python
from odoo import models, fields

class Student(models.Model):
    _name = "student.student"

    name = fields.Char()
    age = fields.Integer()
```

---

এটা Database-এ Table তৈরি করবে।

```text
student_student

id

name

age
```

---

### ASCII

```text
Python Class

↓

ORM

↓

SQL Query

↓

PostgreSQL
```

তুমি SQL লেখো না।

Odoo ORM নিজে SQL তৈরি করে।

# এক লাইনে মনে রাখার ট্রিক

| File                  | কাজ                                                    |
| --------------------- | ------------------------------------------------------ |
| `__manifest__.py`     | Module-এর Configuration ও Metadata                     |
| `__init__.py`         | Python Module Import করার Entry Point                  |
| `models.py`           | Business Logic, Model ও Database Mapping (ORM)         |
| `student_view.xml`    | User Interface (Form, List, Menu, Action)              |
| `controllers.py`      | HTTP Request/Response Handle করে (Website/API-এর জন্য) |
| `ir.model.access.csv` | কে Read/Write/Create/Delete করতে পারবে তা নির্ধারণ করে |

## Interview Tip

যদি কেউ জিজ্ঞেস করে, **"Odoo-তে একটি Module Install হলে কী কী হয়?"**, তাহলে এই ক্রমটি বলবে:

```text
Install Module
      │
      ▼
1. Read __manifest__.py
      │
      ▼
2. Import Python (__init__.py → models)
      │
      ▼
3. Register Models with ORM
      │
      ▼
4. Create/Update Database Tables
      │
      ▼
5. Load Security Rules
      │
      ▼
6. Load XML Data (Views, Menus, Actions)
      │
      ▼
7. Module Ready for Users
```

এই Lifecycle বুঝতে পারলে Odoo Module কীভাবে কাজ করে, তার পুরো ছবিটা পরিষ্কার হয়ে যায়।
