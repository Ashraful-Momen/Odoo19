# STEP 3 â†’ Learn XPath (MOST IMPORTANT)

This is where most developers struggle.

Example:

```xml
<record id="sale_order_form_inherit" model="ir.ui.view">
    <field name="name">sale.order.form.inherit</field>
    <field name="model">sale.order</field>

    <field name="inherit_id" ref="sale.view_order_form"/>

    <field name="arch" type="xml">

        <xpath expr="//field[@name='partner_id']" position="after">
            <field name="customer_code"/>
        </xpath>

    </field>
</record>
```

This means:

* inherit existing form
* locate `partner_id`
* insert new field after it

You MUST master:

* `position="after"`
* `before`
* `inside`
* `replace`
* `attributes`

---

# STEP 4 â†’ Learn Common XPath Patterns

These appear daily in enterprise ERP.

## Add field after another field

```xml
<xpath expr="//field[@name='name']" position="after">
    <field name="mobile"/>
</xpath>
```

---

## Hide field

```xml
<xpath expr="//field[@name='email']" position="attributes">
    <attribute name="invisible">1</attribute>
</xpath>
```

---

## Add page in notebook

```xml
<xpath expr="//notebook" position="inside">

    <page string="Extra">
        <group>
            <field name="notes"/>
        </group>
    </page>

</xpath>
```
