হ্যাঁ, Odoo 17 থেকে 18/19 এ `view.xml` এ অনেক syntax modernize করা হয়েছে। বিশেষ করে `attrs`, `states`, `tree` view, visibility logic — এগুলোর syntax change হয়েছে।
তুমি যদি পুরোনো Odoo 14/15/16 দেখে শিখো, তাহলে Odoo 19 এ এসে অনেক code invalid হবে।

নিচে Odoo 19 view XML এর important changes list দিলাম।

---

# Odoo 19 View XML Important Changes

---

# 1. `tree` → `list`

## Old (Odoo <=16)

```xml
<tree>
    <field name="name"/>
</tree>
```

## New (Odoo 17/18/19)

```xml
<list>
    <field name="name"/>
</list>
```

✅ `tree` tag officially replaced by `list`

---

# 2. `attrs` removed

আগে dynamic invisible/readonly/required সব `attrs` দিয়ে হতো।

## Old

```xml
<field name="amount"
       attrs="{'invisible': [('state','=','done')]}"/>
```

## New

```xml
<field name="amount"
       invisible="state == 'done'"/>
```

✅ এখন direct python-like expression ব্যবহার হয়।

---

# 3. `states` removed

## Old

```xml
<button name="confirm"
        states="draft"/>
```

## New

```xml
<button name="confirm"
        invisible="state != 'draft'"/>
```

---

# 4. Invisible syntax changed

## Old

```xml
attrs="{'invisible':[('type','=','sale')]}"
```

## New

```xml
invisible="type == 'sale'"
```

---

# 5. Readonly syntax changed

## Old

```xml
attrs="{'readonly':[('state','=','done')]}"
```

## New

```xml
readonly="state == 'done'"
```

---

# 6. Required syntax changed

## Old

```xml
attrs="{'required':[('partner_id','!=',False)]}"
```

## New

```xml
required="partner_id != False"
```

---

# 7. Multiple conditions syntax

## Old

```xml
attrs="{
    'invisible':[
        '|',
        ('state','=','done'),
        ('amount','=',0)
    ]
}"
```

## New

```xml
invisible="state == 'done' or amount == 0"
```

---

# 8. AND condition

## Old

```xml
attrs="{
    'readonly':[
        ('state','=','done'),
        ('user_id','!=',uid)
    ]
}"
```

## New

```xml
readonly="state == 'done' and user_id != uid"
```

---

# 9. Notebook/page visibility

## Old

```xml
<page string="Extra"
      attrs="{'invisible':[('type','=','draft')]}">
```

## New

```xml
<page string="Extra"
      invisible="type == 'draft'">
```

---

# 10. Group visibility simplified

## Old

```xml
<group attrs="{'invisible':[('is_company','=',False)]}">
```

## New

```xml
<group invisible="not is_company">
```

---

# 11. Boolean check simplified

## Old

```xml
attrs="{'invisible':[('active','=',False)]}"
```

## New

```xml
invisible="not active"
```

---

# 12. One2many/List view syntax updated

## Old

```xml
<field name="order_line">
    <tree editable="bottom">
        <field name="product_id"/>
    </tree>
</field>
```

## New

```xml
<field name="order_line">
    <list editable="bottom">
        <field name="product_id"/>
    </list>
</field>
```

---

# 13. Conditional button visibility

## Old

```xml
<button name="action_done"
        attrs="{'invisible':[('state','!=','draft')]}"/>
```

## New

```xml
<button name="action_done"
        invisible="state != 'draft'"/>
```

---

# 14. Column invisible syntax

## Old

```xml
<field name="price"
       attrs="{'column_invisible':[('parent.state','=','done')]}"/>
```

## New

```xml
<field name="price"
       column_invisible="parent.state == 'done'"/>
```

---

# 15. Decoration syntax mostly same

এটা খুব বেশি change হয়নি।

```xml
<list decoration-danger="state == 'cancel'">
```

---

# 16. XPath positions same but stricter

Still:

```xml
<xpath expr="//field[@name='partner_id']"
       position="after">
```

কিন্তু Odoo 19 এ xpath miss হলে error handling stricter হয়েছে।

---

# 17. `domain` expression cleaner

## Old

```xml
domain="[('partner_id','=',partner_id)]"
```

## New (still works)

```xml
domain="[('partner_id', '=', partner_id)]"
```

কিন্তু expression parser smarter হয়েছে।

---

# 18. Context syntax cleaner

```xml
context="{'default_partner_id': partner_id}"
```

এখনও same।

---

# 19. Owl frontend integration

Odoo 19 এ অনেক view internally OWL based।

তাই:

* dynamic rendering
* modifiers
* visibility
* reactive behavior

আগের তুলনায় different behave করতে পারে।

---

# 20. Modifier system changed internally

আগে:

```xml
attrs=""
```

backend modifier JSON এ convert হতো।

এখন:

```xml
invisible=""
readonly=""
required=""
```

direct expression evaluate হয়।

এটা সবচেয়ে বড় architectural change।

# সবচেয়ে Important Migration Rule

```

```

এটা ৯০% জায়গায় লাগবে।

# Biggest Odoo 19 Breaking Changes Summary

| Old                | New                         |
| ------------------ | --------------------------- |
| tree               | list                        |
| attrs              | invisible/readonly/required |
| states             | invisible condition         |
| domain tuple logic | python-like expression      |
| modifier dict      | direct attributes           |
