```xml
<record id="view_sale_order_form_custom" model="ir.ui.view">
```

### ব্যাখ্যা:

* `<record>` → Odoo-তে নতুন একটি database record তৈরি করে।
* `id="view_sale_order_form_custom"`
  → এই view-এর unique XML ID।
  → পরে inheritance বা reference করার জন্য ব্যবহার হবে।
* `model="ir.ui.view"`
  → এই record টি `ir.ui.view` model এ save হবে।
  → `ir.ui.view` হলো Odoo এর view management model।

---

```xml
    <field name="name">sale.order.form.custom</field>
```

### ব্যাখ্যা:

* `field name="name"`
  → view-এর internal নাম।
* `sale.order.form.custom`
  → developer friendly name।
  → UI তে user সাধারণত এটা দেখে না।

---

```xml
    <field name="model">sale.order</field>
```

### ব্যাখ্যা:

* এই view কোন model এর জন্য কাজ করবে সেটা বলে।
* `sale.order`
  → Sales Order model।
* অর্থাৎ এই form view Sales Order screen এ ব্যবহার হবে।

---

```xml
    <field name="arch" type="xml">
```

### ব্যাখ্যা:

* `arch`
  → architecture এর short form।
* এখানে পুরো UI XML structure লেখা হয়।
* `type="xml"`
  → content XML format এ আছে।

---

```xml
        <form string="Sales Order">
```

### ব্যাখ্যা:

* `<form>`
  → এটা একটি Form View শুরু করছে।
* `string="Sales Order"`
  → form এর title / label।
* User যখন record open করবে তখন এই form দেখবে।

---

```xml
            <sheet>
```

### ব্যাখ্যা:

* `<sheet>`
  → Odoo form এর main white content area।
* সাধারণত সব form field `sheet` এর ভিতরে থাকে।
* এটা UI কে সুন্দর layout দেয়।

---

```xml
                <group>
```

### ব্যাখ্যা:

* `<group>`
  → field গুলোকে group করে।
* Automatically two-column layout তৈরি করে।
* Form সুন্দরভাবে align হয়।

---


| `<sheet>`    | Main form area        |

# এখানে কোন Model + View সম্পর্ক হচ্ছে?

| XML অংশ                                  | Meaning                    |
| ---------------------------------------- | -------------------------- |
| `model="ir.ui.view"`                     | এটা view record            |
| `<field name="model">sale.order</field>` | এই view sale.order এর জন্য |
| `<field name="partner_id"/>`             | sale.order model এর field  |

# Important Concept

Odoo View = Database field এর UI representation

মানে:

* Python model → database structure
* XML view → frontend UI

# Real Flow

```python

    _name = "sale.order"

    partner_id = fields.Many2one("res.partner")
    date_order = fields.Date()
```

```xml
<field name="partner_id"/>
<field name="date_order"/>
```

# Interview Question Type

## Q: `arch` কি?

Answer:

> Odoo view architecture XML যেখানে UI structure define করা হয়।

---

## Q: `ir.ui.view` কি?

Answer:

> Odoo এর built-in model যা সব view store করে।

---

## Q: `<field>` tag কি করে?

Answer:

> Model এর database field কে UI তে render করে।
