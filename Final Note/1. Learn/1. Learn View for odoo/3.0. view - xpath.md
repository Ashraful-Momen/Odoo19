```xml
<record id="sale_order_form_inherit" model="ir.ui.view">
```

### এই লাইনের কাজ

এখানে নতুন একটি View Record তৈরি করা হচ্ছে।

* `<record>` → Odoo database এ একটি record define করে।
* `id="sale_order_form_inherit"`
  → এই view এর unique XML ID।
  পরে অন্য জায়গা থেকে এই ID দিয়ে reference করা যাবে।
* `model="ir.ui.view"`
  → বোঝাচ্ছে এই record টি `ir.ui.view` model এর মধ্যে save হবে।
  `ir.ui.view` হচ্ছে Odoo এর সব form/tree/kanban/search view এর model।

---

```xml
<field name="name">sale.order.form.inherit</field>
```

### এই লাইনের কাজ

এটা view এর internal নাম।

* `name="name"` → `ir.ui.view` model এর `name` field।
* `sale.order.form.inherit` → view এর readable/internal name।

এটা mainly developer বুঝার জন্য।

---

```xml
<field name="model">sale.order</field>
```

### এই লাইনের কাজ

এই view কোন model এর জন্য কাজ করবে সেটা define করছে।

* `sale.order`
  → Sales Order model।

মানে এই view sales order form modify করবে।

---

```xml
<field name="inherit_id" ref="sale.view_order_form"/>
```

### এই লাইনের কাজ

এখানে existing একটা view inherit করা হচ্ছে।

* `inherit_id`
  → কোন parent/base view inherit করবে।
* `ref="sale.view_order_form"`
  → `sale` module এর original Sales Order Form View।

অর্থাৎ:

👉 Odoo এর default Sales Order Form কে modify করা হচ্ছে।

---

```xml
<field name="arch" type="xml">
```

### এই লাইনের কাজ

এখানে actual XML modification লেখা হয়।

* `arch`
  → architecture অর্থাৎ view structure।
* `type="xml"`
  → XML syntax ব্যবহার করা হচ্ছে।

---

```xml
<xpath expr="//field[@name='partner_id']" position="after">
```

### এই লাইনের কাজ

এখানে XPath দিয়ে existing view এর কোন জায়গায় change হবে সেটা select করা হচ্ছে।

---

## XPath Breakdown

### `//field`

মানে সব `field` tag এর মধ্যে search করো।

---

### `[@name='partner_id']`

যে field এর `name="partner_id"` আছে সেটাকে target করো।

`sale.order` form এ `partner_id` সাধারণত Customer field।

অর্থাৎ:

👉 Customer field খুঁজে বের করো।

---

## `position="after"`

মানে:

👉 selected field এর পরে নতুন কিছু add করো।

Possible positions:

| Position   | Meaning               |
| ---------- | --------------------- |
| before     | আগে add করবে          |
| after      | পরে add করবে          |
| inside     | ভিতরে add করবে        |
| replace    | replace করবে          |
| attributes | attribute modify করবে |

---

```xml
<field name="customer_code"/>
```

### এই লাইনের কাজ

নতুন field add করা হচ্ছে।

* `customer_code`
  → model এ existing field হতে হবে।

এটা `partner_id` field এর পরে show হবে।

---

```xml
</xpath>
```

`arch` field শেষ।


# পুরো Flow টা সহজ ভাষায়

এই XML বলছে:

> Sales Order Form inherit করো
> Customer (`partner_id`) field খুঁজে বের করো
> তারপর তার পরে `customer_code` field add করো।

# Important Things

## 1. `customer_code` field model এ থাকতে হবে

Python model এ field define না থাকলে error দিবে।

Example:

```python
customer_code = fields.Char(string="Customer Code")
```

---

## 2. XPath ভুল হলে view load হবে না

যদি:

```xml
//field[@name='partner_id']
```

না পায়, তাহলে error আসতে পারে।
